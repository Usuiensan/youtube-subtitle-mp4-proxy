できます。おすすめ構成はこれです。

```text
自宅PC Windows 11
  FastAPI app: http://127.0.0.1:8000
  cloudflared: Cloudflare Tunnelで外へ公開

外部URL
  https://video.example.com
    ↓ Cloudflare HTTPS
  Cloudflare Tunnel
    ↓
  http://127.0.0.1:8000
```

ルーターのポート開放は不要です。Cloudflare Tunnel は `cloudflared` から Cloudflare 側へ外向き接続する方式なので、自宅IPを直接公開しません。Cloudflare公式でも、public hostname を `http://localhost:8080` のようなローカルサービスへ紐づける構成が説明されています。

**1. Windows側に必要なものを入れる**
PowerShell で：

```powershell
winget install --id Git.Git
winget install --id Python.Python.3.12
winget install --id DenoLand.Deno
winget install --id Cloudflare.cloudflared
```

FFmpeg はこれで入るか確認：

```powershell
winget search ffmpeg
```

候補が出たら、たとえば：

```powershell
winget install --id Gyan.FFmpeg
```

確認：

```powershell
git --version
python --version
ffmpeg -version
deno --version
cloudflared --version
```

**2. アプリを自宅PCで起動**
任意の作業フォルダで：

```powershell
git clone git@github.com:OWNER/REPO.git youtube-mp4-proxy
cd youtube-mp4-proxy
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
pip install -r requirements.txt "yt-dlp[default]"
```

一時的に環境変数を設定：

```powershell
$env:CACHE_DIR="$PWD\.cache"
$env:DEFAULT_LANG="ja"
$env:MAX_DURATION_SECONDS="1800"
$env:MAX_HEIGHT="720"
$env:YTDLP_EXTRA_ARGS="--js-runtimes deno --remote-components ejs:npm"
```

起動：

```powershell
uvicorn app.main:app --host 127.0.0.1 --port 8000 --proxy-headers --forwarded-allow-ips="127.0.0.1"
```

別PowerShellで確認：

```powershell
curl http://127.0.0.1:8000/healthz
```

**3. Cloudflareでサブドメインを作る**
Cloudflare Dashboard で：

```text
Zero Trust
→ Networks
→ Tunnels
→ Create a tunnel
→ Cloudflared
→ 名前: youtube-mp4-home など
→ Windows を選択
→ 表示された cloudflared service install ... コマンドを管理者PowerShellで実行
```

次に Public Hostname を追加：

```text
Subdomain: video
Domain: example.com
Type: HTTP
URL: localhost:8000
```

これで：

```text
https://video.example.com
```

が自宅PCの `http://localhost:8000` へつながります。外側のHTTPSはCloudflareが担当します。

**4. 外から確認**

```powershell
curl https://video.example.com/healthz
```

HLS：

```powershell
curl -L https://video.example.com/youtube-hls/SpQZyPBAtA0/ja
```

**重要**
公開するなら `API_KEY` を必ず入れたほうがいいです。入れないとURLを知っている人が変換を走らせられます。

あと、自宅PCがスリープ/電源OFFだと当然使えません。Windowsの電源設定でスリープ無効、または使用時だけ起動する運用になります。
